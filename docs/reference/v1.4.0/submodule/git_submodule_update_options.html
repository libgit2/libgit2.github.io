git_submodule_update_options

Version:

Submodule update options structure

Initialize with GIT_SUBMODULE_UPDATE_OPTIONS_INIT. Alternatively, you can use git_submodule_update_options_init.

Signature

GitHub
typedef struct git_submodule_update_options { unsigned int version git_checkout_options checkout_opts git_fetch_options fetch_opts int allow_fetch };

Members

unsigned int
version
git_checkout_options
checkout_opts

These options are passed to the checkout step. To disable checkout, set the checkout_strategy to GIT_CHECKOUT_NONE. Generally you will want the use GIT_CHECKOUT_SAFE to update files in the working directory.

git_fetch_options
fetch_opts

Options which control the fetch, including callbacks.

The callbacks to use for reporting fetch progress, and for acquiring credentials in the event they are needed.

int
allow_fetch

Allow fetching from the submodule's default remote if the target commit isn't found. Enabled by default.
